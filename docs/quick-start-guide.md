<h1><a id="user-content-getting-started-guide" class="anchor" aria-hidden="true" href="#getting-started-guide"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Getting Started Guide</h1>
<p>Getting your app up and running on Firebolt® takes just a few simple steps.</p>
<p>Most of your app's code is dealing with your own content and services, and rightfully so! However, there are a few minor APIs that your app will need to call in order to function in a Firebolt®-compliant manner. Most of these APIs require just one line of code.</p>
<p>Firebolt® has zero dependencies, and leverages industry standard ECMAScript, which is compiled into your app, and packaged up using whatever JavaScript packaging strategy you already use.</p>
<p>In order to get your app up and running on Firebolt®, you'll need to:</p>
<ul>
<li>Install the Firebolt® package via your package manager (i.e. <code>npm</code>)</li>
<li>Import a few Firebolt® modules into your code-base</li>
<li>Call a few simple methods to allow Firebolt® and your app to communicate</li>
</ul>
<h2><a id="user-content-installing-firebolt" class="anchor" aria-hidden="true" href="#installing-firebolt"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installing Firebolt®</h2>
<p>Installing Firebolt® is simple, because it's a publicly hosted package at npmjs.com:</p>
<div class="snippet-clipboard-content position-relative overflow-auto" data-snippet-clipboard-copy-content="cd &lt;your project directory&gt;
npm install @firebolt-js/sdk
"><pre><code>cd &lt;your project directory&gt;
npm install @firebolt-js/sdk
</code></pre></div>
<p>Now you're ready to use Firebolt® in your project.</p>
<h2><a id="user-content-importing-modules" class="anchor" aria-hidden="true" href="#importing-modules"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Importing Modules</h2>
<p>As mentioned before, Firebolt® uses standard ECMAScript imports for all of its modules. This allows you to only import the modules you use.</p>
<p>To get your app using Firebolt® modules, just import them.</p>
<p>For example, this will import the Lifecycle and Metrics modules:</p>
<div class="highlight highlight-source-js position-relative overflow-auto" data-snippet-clipboard-copy-content="import { Lifecycle, Metrics } from '@firebolt-js/sdk'
"><pre><span class="pl-k">import</span> <span class="pl-kos">{</span> <span class="pl-v">Lifecycle</span><span class="pl-kos">,</span> <span class="pl-v">Metrics</span> <span class="pl-kos">}</span> <span class="pl-k">from</span> <span class="pl-s">'@firebolt-js/sdk'</span></pre></div>
<h2><a id="user-content-starting-up-your-app" class="anchor" aria-hidden="true" href="#starting-up-your-app"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Starting up your app</h2>
<p>The first thing your app will want to do is tell Firebolt® that it's ready. Firebolt® avoids assuming your app is minimally usable, and gives you complete control over when your app is ready to be presented to end users.</p>
<p>When you've rendered enough of your app just call 'ready':</p>
<div class="highlight highlight-source-js position-relative overflow-auto" data-snippet-clipboard-copy-content="Lifecycle.ready()
"><pre><span class="pl-v">Lifecycle</span><span class="pl-kos">.</span><span class="pl-en">ready</span><span class="pl-kos">(</span><span class="pl-kos">)</span></pre></div>
<p>That's it! Now Firebolt® knows that it's safe to present your app to end users, and can do so as quickly as possible.</p>
<p>It's recommended to also listen to any Firebolt® events <em>before</em> calling ready, to ensure you don't miss any.</p>
<p>See <a href="/firebolt-docs/authored-docs/blob/quick-start-guide/api/Lifecycle/#ready">Lifecycle</a> for more info.</p>
<h2><a id="user-content-reporting-errors" class="anchor" aria-hidden="true" href="#reporting-errors"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Reporting Errors</h2>
<p>In order to track the quality of both your app and the Firebolt® experience, Firebolt® needs to know when the user experiences errors in your app. Whenever an error occurs, simply pass the error to the Firebolt® Metrics API. For example, to report a 404 network error when loading a required asset:</p>
<div class="highlight highlight-source-js position-relative overflow-auto" data-snippet-clipboard-copy-content="Metrics.error('network', '404', true, 'The requested asset could not be found')
"><pre><span class="pl-v">Metrics</span><span class="pl-kos">.</span><span class="pl-en">error</span><span class="pl-kos">(</span><span class="pl-s">'network'</span><span class="pl-kos">,</span> <span class="pl-s">'404'</span><span class="pl-kos">,</span> <span class="pl-c1">true</span><span class="pl-kos">,</span> <span class="pl-s">'The requested asset could not be found'</span><span class="pl-kos">)</span></pre></div>

<h2><a id="user-content-reporting-content-minutes" class="anchor" aria-hidden="true" href="#reporting-content-minutes"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Reporting Content Minutes</h2>
<p>In order to track frequency of errors, Firebolt® needs to know which spans of time your user is watching content.</p>
<p>To track this, call the <code>startContent</code> method before the user starts watching content:</p>
<div class="highlight highlight-source-js position-relative overflow-auto" data-snippet-clipboard-copy-content="Metrics.startContent()
"><pre><span class="pl-v">Metrics</span><span class="pl-kos">.</span><span class="pl-en">startContent</span><span class="pl-kos">(</span><span class="pl-kos">)</span></pre></div>
<p>See <a href="/firebolt-docs/authored-docs/blob/quick-start-guide/api/Metrics/#startcontent">Metrics.startContent</a> for more info.</p>
<p>Note that no identifiers are required for either method, as Firebolt® only tracks the length of engagement, not specifically <em>what</em> the user is watching.</p>
<p>When the user is done watching content, call <code>stopContent</code>:</p>
<div class="highlight highlight-source-js position-relative overflow-auto" data-snippet-clipboard-copy-content="Metrics.stopContent()
"><pre><span class="pl-v">Metrics</span><span class="pl-kos">.</span><span class="pl-en">stopContent</span><span class="pl-kos">(</span><span class="pl-kos">)</span></pre></div>
<p>See <a href="/firebolt-docs/authored-docs/blob/quick-start-guide/api/Metrics/#stopcontent">Metrics.stopContent</a> for more info.</p>
<h2><a id="user-content-inspecting-device-capabilities" class="anchor" aria-hidden="true" href="#inspecting-device-capabilities"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Inspecting Device Capabilities</h2>

<p>Firebolt® provides a number of convenient getters that you are welcome to use for this purpose, such as 'video resolution' as displayed below:</p>
<div class="highlight highlight-source-js position-relative overflow-auto" data-snippet-clipboard-copy-content="let [width, height] = await Device.videoResolution()
